# Claim Boundaries

Claim boundaries prevent visual proof from becoming product overclaiming.

## Required Pattern

Each artifact should include:

- what was tested
- what worked
- what did not work
- what this proves
- what this does not prove
- current blockers
- recommended next action

## Examples

- A plugin editor screenshot proves the editor was visible, not that a workflow completed.
- A screenshot walkthrough proves a visual narrative was generated from artifacts, not that the workflow was live-recorded.
- A backend route response proves the backend state exists, not that the live UI consumed or rendered it.
- Exact proof for one supported subset does not prove full reconstruction.
- A proof in one host does not prove the same behavior in another host.
